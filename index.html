How the Web Works

The web is a bunch of computers that talk to each other. When a person goes to a web page, their computer sends a HTTP Request to a server. The server finds a HTML doc and sends it back to the computer where a web browser receives the page and displays it on the screen.

HTML: Hypertext Markup Language

HTML documents are the majority of the web contents. These docs contain text which reflect what you see and how it looks on the screen.

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. This video explains the distinction well.

Types of Tags:
  1. B - Makes the text content bold. 
  2. EM - Makes the text content initialized. 
  3. Break - Makes one line break, has no closing tag. 
  4. Paragraph- Creates an "invisible box" around the text content. 
  5. Anchor - Makes a link to another page. 
  6. Img - Tags an image.
  7. Div - Makes a box around content which allows for markup of the content. It has no required attributes, but style and class are common. Because it is a block level element, the browser will display line breaks before and after it.
  8. Span - Allows for markup of a single line of element. no required attributes, but style and class are common.
Why Computers are Stupid

Computers are stupid because they literally interpret instructions. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

Inline vs Block Elements

Inline: Inline elements are normally displayed without line breaks. Note: Only (br) ends a line, then wraps to the next line (e.g. a, span, br, img, strong)
Block: Creates an invisible box around the text. It can have height and width. Normally start (and end) with a new line, when displayed in a browser. (e.g. Div, p, form)

Tree-like Structure

A webpage is made up like a house. HTML is like the walls of the house. CSS controls how the house looks, like the color of the walls. Javascript shapes how one interacts with the house, like a garage opener. Web browser is using a tree structure to determine how to display pages.

HTML and CSS are both "languages"•HTML controls the "structure" of a web page:
- CSS controls the "style" of a page (how it looks)
- When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page.

DOM - Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

Boxes

Boximizing Process: After the dom is formed, everything on the webpage tend to be in boxes or boxes inside boxes and its box has its own class name. The tag name div is used to define a box.
Note: Boxing makes it easy to manipulate or deal with different images, text, multimedia. The more the boxes the more the webpage get crouded.

Tools

There are 3 developer tools to write HTML documents:
1. Scratchpad:This gives a real time feedback of how your webpage will look; can be used for simple webpages.
2. Codepen: This takes 2-3 seconds to give feedback of how your page will look like.
3. Sublime: This need one to refresh the page, but it is a more professional tool for web developers.

Lesson 3: Repetition & CSS

Avoiding repetition is important for a variety of reasons.

Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears, problems will arise. Instead of manually giving each one the same style, I was able to assign each lesson div the same class and then specify the style for that class in only one place! By avoiding repetion, it's now very easy for me to make changes to this style.

CSSstands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.
